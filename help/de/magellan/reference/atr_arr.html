ATR, ARR, Tooltips und Regionsinfo

Funktionsweise

Das so genannte Ersetzersystem ist letztlich eine Art von Programmiersprache, die es erlaubt, aus den Infos über Regionen bestimmte Werte auszurechnen. Das Ersetzersystem wird für den ATR und ARR verwendet, sowie für die Tooltips der Karte und die Regionskurzinfos in der Detailanzeige. Der ATR dient zur Beschriftung der Karte. Mit dem ARR kann man Regionen anhand eines Zahlenwertes und einer Farbskala einfärben. Natürlich kann man auch ATR, ARR und Tooltip kombinieren. Dann erhält man farbige Regionen und passende Texte auf jeder Region mit optionalen Zusatzinfos. Auch die ersten Zeilen der Detailanzeige können mit Hilfe dieser Sprache konfiguriert werden. Die folgende Abbildung zeigt eine Karte, die anhand der Handelsgüter eingefärbt und beschriftet ist.

Welche Ersetzungsmöglichkeiten gibt es? Folgende Wörter sind im Zusammenhang mit den Tooltips wichtig. In Klammern ist jeweils ein Beispiel angegeben. Es gibt

  • Variablen (§herb).
  • zusammengesetzte Variablen (§item§Speer).
  • Bedingungen (§if§not§isOzean§text1§else§text2§end§).
  • zusammengesetzte Schalter (§faction§abcd). Beschränken die Funktion der auf sie folgenden Ersetzer nach festlegbaren Kriterien
  • Rechenoperationen (§+§3§2)
  • Strings (§beliebiger Text)

Eine Definitionszeichenkette (Zeile) besteht nun aus mehreren hintereinander geschriebenen Wörtern, denen jeweils das Trennzeichen § vorangestellt ist.

Einfache Beispiele

Beispiel: Text mit Variable

§Kraut §herb führt in einer Region, wo das Kraut Elfenlieb wächst, zur Ausgabe Kraut Elfenlieb . Wenn Magellan das Wort erkennt (hier "herb"), dann wird es in jeder Region durch den entsprechenden Wert ersetzt (hier "Elfenlieb"). Andernfalls wird einfach der Text ausgegeben (hier "Kraut").

Beispiel: Zählen von Gegenständen

Mit der zusammengesetzten Variable §item§Gegenstand kann man sich die Anzahl von Gegenstand anzeigen lassen. §item§Speer gibt die Anzahl Speere in dieser Region aus. §item§Speer§ §item§Armbrust ergibt Anzahl der Speere gefolgt von einem Leerzeichen gefolgt von der Anzahl Armbrüste. Der Gegenstand muss genau so geschrieben werden, wie er in Magellan genannt wird.

Beispiel: Anzeige nicht auf Ozeanen

Mit if kann man die Ausgabe einschränken. So sorgt §if§not§isOzean§Bäume §trees dafür, dass nur in Nicht-Ozeanregionen die Anzahl Bäume angezeigt wird. Das sorgt bei Dingen wie Bäumen für mehr Übersichtlichkeit, da dann nicht in jedem Ozeanfeld das Wort Bäume steht. Das §not kann man auch weglassen. Dann wird nur in Ozeanregionen angezeigt (was mit Bäumen natürlich keinen Sinn ergibt). Die komplette Syntax lautet §if[§not]§{Bedingung}§{wenn Bedingung wahr}§else§{wenn Bedingung falsch}§end.
Dabei ist not optional für die Verneinung. Die geschweiften Klammern inklusive Inhalt sollen durch die gewünschten Ausgaben ersetzt werden.

Beispiel: Zählen auf Einheiten von einer Partei beschränken

Der zusammengesetzte Schalter §faction§Parteinummer beschränkt das Zählen von Gegenständen auf die angegebene Partei. In einer Region, wo die Partei abcd 3 Speere hat, führt §Speer §faction§abcd§item§Speer zur Ausgabe von Speer 3

Beispiel: befreundete und alle Personen

Der zusammengesetzte Schalter §priv§Vertrauenslevel beschränkt das Zählen auf Parteien mit dem angegebenen Vertrauenslevel. §priv§clear hebt diese Einschränkung wieder auf. Parteien, bei denen das Passwort gesetzt ist, haben Vertrauenslevel 100. Ein HELFE ALLES entspricht dem Wert 60. Möchte man eigene und fremde Personen zählen kann man folgendes verwenden: §priv§100§Eigene §count§priv§clear§ | Alle §count .

Beispiel: Addieren

Geschrieben wird nach der polnischen Notation, d.h. die Rechenoperatoren stehen vor den beiden (oder mehr) Operanden. §+§item§Speer§item§Armbrust addiert Speere und Armbrüste zusammen. Durch diese Schreibweise benötigt man keine Klammern.
a * (b + c) ist in polnischer Notation * a + b c, also §*§a§+§b§c oder auch §*§+§b§c§a
a * b + c lautet + * a b c, also §+§*§a§b§c oder auch §+§c§*§a§b.

Hinweise und Stolpersteine

  • Man sollte darauf achten, dass am Ende der Zeile keine Leerzeichen sind. Diese verhindern das korrekte Erkennen der Ersetzer. Am besten macht man zu Beginn und Ende der Zeile je ein §. Das verhindert Fehler. Die Syntax erlaubt jedoch auch das Weglassen von § am Anfang und Ende. Nur das Wort herb funktioniert also auch.
  • Bei Tooltips und in den Regionsinfos kann HTML benutzt werden. Dazu muss die ganze Ausgabe in HTML-Tags eingeschlossen sein, also zum Beispiel <html><body><b>§rname§</b> <i>§herb§</i> </body></html>.
  • §newline erzeugt einen Zeilenwechsel im ATR. Einen Tooltip schreibt man jedoch besser komplett als HTML oder verwendet ein Leerzeichen, anstelle des newline.
  • Beim ARR muss das Ergebnis des gesamten Ausdrucks eine Zahl sein. Jeder String verhindert eine richtige Ausgabe. Dazu gehören auch Zeilenumbrüche und Leerzeichen.
  • Wenn die Syntax falsch ist, erhält man keine Fehlermeldung, sondern nur eine falsche Ausgabe.
  • Ob die Ausgaben des ARR richtig sind, prüft man am besten mit einem entsprechenden Tooltip.
  • Ein Ersetzer, der etwas zählt (z.B. Speere), tut dies für alle Einheiten in einer Region. Möchte man dies auf z.B. eigene Einheiten einschränken, so müssen einschränkende Ersetzer wie priv oder faction verwendet werden.
  • Ein §clear hinter faction oder priv hebt die Einschränkung wieder auf.
  • Ein Leerzeichen erhält man, indem man hinter § ein Leerzeichen setzt. z.B. §Ersetzer§ §Ersetzer
  • Möchte man zum Beispiel nicht den Ersetzer herb aufrufen, sondern die Zeichenkette 'herb' ausgeben, kann man das so erreichen: §\herb§. Einen Backslash '\' kann man so ausgeben: §\§

Ersetzer Erläuterung - wenn nichts anderes angegeben, gilt der Ersetzer für eine Region. Parameter müssen genau so angegeben werden, wie sie Magellan anzeigt.
